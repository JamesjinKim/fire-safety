Overview

비즈니스 성공을 위한 영리한 세그먼트 차별화

기존의 무거운 통합 ERP 시장의 틈새를 찾아, 본질인 '자체점검 일지 및 보고서 작성'의 가치를 10배로 극대화합니다.

기존의 무거운 ERP (실패 모델)

회피 대상 1순위, 거대한 만능 시스템

수많은 중소 점검업체들이 값비싼 라이선스를 내고도 복잡한 UI와 과도한 의무 기능 때문에 도입을 회피하는 영역입니다.

  • ✕ 다단계 복잡한 전자결재 프로세스
  • ✕ 무거운 회사 인사/근태/급여 관리 연동
  • ✕ 사용률 5% 미만의 재고 관리 기능 내장
  • ✕ 점검 데이터의 시계열 변조 추적 불가

단계 A: 사무실 (오류 최소화)

공공데이터 기반 건축물 정보 파싱

도로명 주소 입력 시, 공공 데이터베이스(건축물대장 OpenAPI)와 즉시 연동하여 사용승인일, 연면적, 층수, 동수, 구조 등 법령 계산에 필요한 핵심 수치를 휴먼 에러 없이 자동으로 로딩합니다.

단계 A: 사무실 (Immutable)

룰 엔진 기반 소방법령 산출 및 지시서 동결

LLM의 환각(Hallucination) 위험을 배제한 100% 하드코딩 룰 매칭 엔진이 건물의 사용승인년도 시점의 소방법 및 시설 고시 조문을 역추적 매칭합니다. 해당 건물에만 활성화해야 할 점검표 종류를 골라낸 뒤 해시 암호화를 통해 지시서(WorkOrder)를 단단히 잠금(Freeze) 처리합니다.

단계 B: 현장 (실시간 기록)

작업지시 기준 불일치(Discrepancy) 기록

현장 작업자는 동결된 작업지시를 보며 점검을 수행합니다. 지시서에 기입된 가상 소방 기구 수량과 실제 현장 설비 현황이 불일치하는 경우 즉시 수정하며, 이 모든 내역은 삭제가 불가능한 변경로그(Append-only 로그) 형태로 완벽 기록되어 보고서 신뢰성을 지킵니다.

// Append-only Discrepancy Log
{
  "occurredAt": "2026-05-24T08:24:00Z",
  "inspector": "김대리(ID: 2849)",
  "fieldPath": "facilities[0].quantity",
  "before": 24, // 지시서 소화기수
  "after": 28,  // 실제 실측수
  "reason": "4층 발코니 신설 구역 추가 설치 확인"
}
출력: 인쇄 & 정산

법정양식 A4 1:1 정밀 픽셀 렌더링

Product Timeline

스마트 소방안전점검 로드맵

안정적인 코어 시스템 v1 빌드를 시작으로, 단계적으로 지능형 모바일 확장 및 AI 혁신 모델로 발전해 나갑니다.

Phase 1 (v1.0 POC)

결정론적 룰 기반 코어 웹폼 및 인쇄 최적화

건축물대장 정보에 근거하여 시계열 소방법과 동적으로 1:1 매칭합니다. A4 정밀 1:1 서식 렌더러와 수동 불일치 변경 로그 추적 기반 다듬기.

Next.js App Router CSS Print Engine Rule Matcher
Phase 2 (v1.5)

모바일 친화도 향상 및 단일 음성 받아쓰기

모바일 브라우저 편의성을 극대화하고 텍스트 입력의 답답함을 해소하기 위해 단일 필드 전용 '음성 메모 쓰기' 마이크 UI 적용.

STT API PWA Mobile UX Field Voice Memo
Phase 3 (v2.0)

카카오톡 텍스트 분석 및 다중 협업 락킹 시스템

동시 다발적 현장 데이터 덮어쓰기 사고를 예방하는 설비별 영역 락킹(Locking)을 적용합니다. 단톡방 텍스트 데이터 추출 ➔ 5원 미만의 보고서 자동 파싱 엔진 배포.

BaaS Section Lock Kakao Talk Parser LLM Extractor
Phase 4 (v3.0)

실시간 AI 스마트 무전 보고 서비스 완성

인앱 그룹 통화 채널을 구축하고, 실시간 무전(Push-to-Talk) 음성을 클라우드 파이프라인으로 처리하여 법정 9호 및 4호서식을 AI가 실시간으로 입력해주는 궁극의 자동화 실현.
