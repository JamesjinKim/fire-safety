## 1. 배경 및 비전

소방안전점검 현장은 열악한 먼지 노출, 고공 작업(사다리), 안전장갑 착용 등으로 인해 **모바일 기기에서 키보드로 텍스트를 직접 입력하기가 극도로 어렵습니다.** 

현장 작업자들이 **"음성이나 일상 대화(단톡방)만으로 소통하면, AI가 현장 메모와 불량 사항 후보를 구조화하고 관리자가 검수해 법정 서식에 반영하는 시스템"**을 완성하여 현장 노동 강도를 낮추는 것을 최종 비전으로 합니다.

이 문서는 프로젝트 초기 버전(v1)에서는 제외된 **"그룹 통화 및 AI 현장기록 구조화"** 기능을 장기적으로 안전하고 저렴하게 구현하기 위한 구체적인 아키텍처와 로드맵을 기록합니다.

> 경계: 이 로드맵의 AI는 현장 메모·불량 사항을 구조화하는 보조 기능이다. 건축년도별 법령 스냅샷 조회, 설비 섹션 추천, 작업지시 동결은 `fire-inspection-system`의 결정론적 룰 엔진과 고객 BYOK 법령 스냅샷 전략을 따른다.

## 2. 핵심 아키텍처 및 데이터 흐름

AI 기반 보고서 완성 기능은 **[수집 ➔ 음성 변환(STT) ➔ AI 구조화(LLM) ➔ HTML 반영]**의 4단계 파이프라인으로 구현됩니다.

```
┌─────────────────┐       ┌─────────────────┐       ┌──────────────────┐
│  현장 음성/텍스트  │ ───▶  │ STT (음성인식)  │ ───▶  │   LLM 구조화     │
│  (무전기 or 카톡) │       │ * 소방용어 매핑  │       │ (비정형 ➔ 구조화) │
└─────────────────┘       └─────────────────┘       └────────┬─────────┘
                                                             │
                                                             ▼ (JSON 변환)
┌─────────────────┐       ┌─────────────────┐       ┌──────────────────┐
│  A4 PDF 인쇄    │ ◀───  │ HTML 양식 반영  │ ◀───  │  Form9 / Form4   │
│  (1:1 정합 출력)  │       │ (Auto-fill)     │       │  데이터 모델 매핑  │
└─────────────────┘       └─────────────────┘       └──────────────────┘
```

### ① 음성 변환 (STT) 단계의 특화 기술
소방 점검 현장에서 쓰이는 전문 용어(*수신기, 감지기, 피난구유도등, 축전지, 오동작* 등)는 일반 범용 STT 엔진에서 오인식되기 쉽습니다.
- **대응책**: 음성 인식 엔진(예: Whisper, Google Cloud Speech-to-Text)에 **소방 전문 용어 사전(Custom Vocabulary)**을 프롬프트 형태로 주입하여 정확도를 극대화합니다.

### ② 비정형 대화의 구조화 (LLM 파싱)
음성에서 텍스트로 변환된 자연어(구어체)를 저비용 LLM으로 데이터 구조(JSON Schema)로 가공합니다. 구체 모델은 출시 시점의 비용·성능 비교로 결정합니다.
- **입력**: *"3층 복도에 있는 소화기가 압력계가 초록색 범위를 벗어나서 아래로 떨어져 있네. 교체해야겠어."*
- **LLM 구조화 출력 (JSON)**:
  ```json
  {
    "location": "3층 복도",
    "checklistCode": "1-A-001", // 소화기구 점검표 코드 매핑
    "status": "defect",
    "defectDetail": "지시압력계 저하 (녹색 범위 이탈)",
    "actionRequired": "소화기 현장 즉시 교체"
  }
  ```

## 3. 극가성비 비즈니스를 위한 AI 비용 최적화 전략

"실시간 그룹 통화 전체를 상시 녹음 및 분석"하는 모델은 API 비용 폭탄과 대용량 데이터 업로드 비용으로 인해 사업성이 무너질 수 있습니다. 따라서 **비용을 100분의 1로 줄이는 극가성비 전략**을 채택합니다.

```
[전체 상시 녹음 방식]  ➔ 2시간 통화 = 점검당 약 3,000원 초과 (사업성 악화)
[무전기식 음성 보고]  ➔ 2분 요약 녹음 = 점검당 약 18원 (초저비용 SaaS 가능)
[단톡방 텍스트 분석]  ➔ 대화방 .txt 업로드 = 점검당 약 5원 (STT 비용 0원)
```

### 전략 A: 무전기식 '필요할 때만 녹음' (Push-to-Talk)
* **방식**: 작업자들은 평소 카카오톡 보이스톡이나 일반 전화로 소통합니다. 보고가 필요한 불량 사항이 있을 때만 앱 내부의 마이크 버튼을 눌러 **5~10초간 짧게 무전하듯 말하고 녹음**합니다.
* **비용 효과**: 1회 점검당 불량 사항 보고가 약 15건 발생할 경우, 총 녹음 시간은 **2분 이내**입니다.
  - STT API 비용: 약 16원 (분당 8원 기준)
  - LLM 구조화 비용: 약 2원 (저비용 모델 기준)
  - **총 원가: 약 18원 / 점검 1건당**

### 전략 B: 카카오톡 단톡방 '텍스트 파일 추출' (STT 비용 0원)
* **방식**: 점검팀 전원이 속한 카카오톡 단톡방을 개설하여 현장에서 *"3층 유도등 불량", "2층 소화전 누수"* 등을 자유롭게 입력합니다. 점검 완료 후 관리자가 **"대화 내용 내보내기(.txt)"**를 통해 텍스트 파일을 올리면 AI가 이를 한 번에 읽어 매핑합니다.
* **비용 효과**: 음성 인식(STT) 과정이 필요 없으므로 API 비용은 오직 LLM 분석 비용만 발생합니다.
  - **총 원가: 약 5원 미만 / 점검 1건당**

## 4. v1 빌드 단계에서의 사전 설계 대비책

장기 로드맵이 v1 개발이 끝난 후 무리 없이 적용되려면, **초기 데이터 구조 설계 단계에서 아래 사항이 반드시 반영되어야 합니다.**

1. **점검 항목의 고유 식별자(ID) 표준화**:
   - 별지 제4호서식의 모든 세부 점검 항목에 고유 식별코드(예: `1-A-001` ~ `32-Z-999`)를 완벽하게 매핑하여 데이터베이스(JSON)를 구축합니다. 향후 AI가 이 코드를 기준으로 데이터를 입력합니다.
2. **`Discrepancy` (변경 사항 로그) 규격화**:
   - 현장에서 작업자가 입력을 수정하거나 새로 등록할 때 생성되는 데이터 구조(`before`, `after`, `reason`)를 표준 JSON Schema로 동결하여 향후 AI가 출력하는 JSON 포맷과 1:1 결합되도록 설계합니다.

## 5. 단계별 진화 로드맵 (Evolution Phases)

```
┌──────────────┐      ┌────────────────┐      ┌────────────────┐      ┌────────────────┐
│  Phase 1     │ ──▶  │  Phase 2       │ ──▶  │  Phase 3       │ ──▶  │  Phase 4       │
│  폼 기반 v1   │      │ 단일 음성 메모  │      │ 카톡 텍스트 분석 │      │ 실시간 무전 AI  │
│  (기본기 다짐)│      │  (STT 폼 연동) │      │ (비동기 파일)  │      │ (실시간 동기화) │
└──────────────┘      └────────────────┘      └────────────────┘      └────────────────┘
```

* **Phase 1: 웹폼 및 인쇄 최적화 (v1)**
  - 수동 웹폼 및 A4 1:1 정합 출력 인쇄 엔진을 완벽히 구축합니다.
* **Phase 2: 단일 필드 '음성 간편 입력' 도입 (v1.5)**
  - 긴 텍스트 입력창 옆에 마이크 버튼을 달아, 탭 한 번으로 음성을 텍스트로 변환해주는 단일 기능(Simple dictation)을 도입하여 현장 거부감을 줄입니다.
* **Phase 3: 카카오톡 단톡방 텍스트 추출 AI 적용 (v2.0)**
  - 카카오톡 단톡방 텍스트(.txt) 혹은 개별 음성 메시지 모음을 파일로 일괄 등록하면 현장 메모와 불량 사항 후보를 구조화한다. 법정 설비 추천과 작업지시 동결은 기존 룰 엔진 결과를 따른다.
* **Phase 4: 실시간 그룹 통화 및 웹RTC 지능형 완성 (v3.0)**
  - 인앱 VoIP 그룹 통화 시스템 및 Push-to-Talk 스트리밍 API를 구축하여 실시간 현장 기록 구조화 흐름을 E2E로 완성합니다.
